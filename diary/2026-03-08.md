#### **1. Corrección de Bugs en RPGService (3 bugs críticos)**

**BUG #1: `add_coins()` fallaba con claves plural**
- **Problema:** Método solo aceptaba `obolo` (singular), fallaba con `obolos` (plural)
- **Solución:** Normalización automática plural → singular con dict `_COIN_NORMALIZE`
- **Archivo:** `src/services/rpg.py`
- **Commit:** `f7898bd`

**BUG #2: Fórmula de nivel incorrecta**
- **Problema:** Usaba `sqrt(xp/100)` en lugar de tabla discreta
- **Solución:** Implementada tabla `LEVEL_XP` con valores específicos por nivel
- **Cambio:**
  ```python
  # ANTES (roto)
  level = int((xp / 100) ** 0.5)
  
  # AHORA (correcto)
  LEVEL_XP = {1: 0, 2: 100, 3: 200, 5: 500, 10: 2000...}
  ```
- **Archivo:** `src/services/rpg.py`
- **Commit:** `f7898bd`

**BUG #3: `log_bad_habit()` solo quitaba Wyrd**
- **Problema:** No aplicaba penalización de monedas según diseño
- **Solución:** Implementada tabla `COIN_PENALTIES` y aplicación automática
- **Ejemplo:**
  ```python
  COIN_PENALTIES = {
      "smoking": {"dracmas": 3},
      "drinking": {"dracmas": 2, "obolos": 5}
  }
  ```
- **Archivo:** `src/services/rpg.py`
- **Commit:** `f7898bd`

#### **2. Sistema de Dados Caronte (NUEVO)**

**Archivo creado:** `src/services/caronte_dice.py` (11KB)

**Funcionalidades:**
- **Tirada diaria:** Dados al final del día según nivel/racha
- **3 Combos detectables:**
  1. **Póker** (3 iguales + comodín) → XP + monedas + artefacto **common**

**Sistema de Dados por Nivel:**
```python

Nivel 10-14: 3 dados, pool mejorado
Nivel 15-19: 4 dados, pool mejorado
Nivel 20-29: 4 dados, pool avanzado
Nivel 30+:   5 dados, pool avanzado
```

**Multiplicadores de Racha:**
- 0-6 días: x1.0
- 7-13 días: x1.2
- 14-20 días: x1.5
- 21-29 días: x1.75
- 30+ días: x2.0

#### **3. Sistema de Indulgencias (NUEVO)**

**Funciones implementadas:**
- `get_indulgence_cost()` - Calcula coste con descuento
- `buy_indulgence_before()` - Compra prevención

- `use_indulgencia()` - Consume token del perfil

## 💭 Notas Técnicas

### **Decisiones de Diseño:**

1. **Groq en lugar de OpenAI:**
   - Más rápido (llama-3.3-70b)
   - Gratis para desarrollo
   - Compatible con OpenAI API

2. **Normalización monedas:**
   - Aceptar plural/singular evita errores frontend
   - Consistencia interna con singular

3. **Tabla LEVEL_XP discreta:**
   - Más control sobre progresión
   - Evita niveles fraccionarios
   - Permite ajustar dificultad por nivel

4. **Indulgencias como tokens:**
   - Columna simple `int` en perfil
   - Log separado para auditoría
   - Flexible para futuras mecánicas

### **Dependencias Groq:**
```bash
pip install groq
# O ya incluido con httpx en requirements.txt
```

### **Variables de entorno necesarias:**
```env
GROQ_API_KEY=gsk_...
DEFAULT_USER_ID=1
DATABASE_URL=sqlite:///./data/life.db
```
